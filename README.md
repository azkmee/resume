# Resume

A living record of my professional accomplishments and resume material.

## Contents

- [`accomplishments-log.md`](./accomplishments-log.md) — STAR-format log of wins, used as source material when updating my CV.
- [`docs/nosql-schema.md`](./docs/nosql-schema.md) — MongoDB schema plan for the resume + portfolio + blog website.
- [`db/`](./db) — runnable scaffold: `$jsonSchema` validators, indexes, seed data, and setup/migration/seed scripts.

## Resume website (MongoDB-backed)

The site is backed by MongoDB. Content is authored by a human and generated by Claude
(written through the MongoDB MCP server), then reviewed and published.

- Data model & rationale: [`docs/nosql-schema.md`](./docs/nosql-schema.md)
- Setup, validators, seeds, scripts: [`db/README.md`](./db/README.md)

Collections: `profile` · `experiences` · `accomplishments` · `projects` · `skills` ·
`education` · `blog_posts` · `media` · `pages` · `testimonials`.

## Conventions

- Entries follow **Situation · Task · Action · Result**.
- Quantify results wherever possible (latency, throughput, $, hours saved, etc.).
- Capture wins as they happen — don't wait for review season.
